- ROS Master  เป็นส่วนที่ทำหน้าที่ลงทะเบียนชื่อ และค้นหาส่วนที่เหลือทั้งหมด ใน computation graph คล้ายทะเบียนราษฎร์ ROS Master จะรู้จักทุกอย่าง ไม่ว่าจะเป็นชื่อ Node ชื่อ request respone ต่างๆว่าอยู่ตรงไหน ถ้าอยากรู้อะรัยให้ถาม ROS Master
ROS Topics เป็นตัวแทนของ node ที่จะสามารถทำการรับ-ส่ง message เข้าไปได้โดย Topics จะเป็นการกำหนดเป็นชื่อเพื่อให้เราสามารถทำการกำหนดต้นทางหรือปลายทางที่ต้องการรับ-ส่ง message โดยจะเป็นการสื่อสารแบบทางเดียว จะมีแค่ publisher กับ subscriber คล้ายทีวี มีแต่คนพูด และมีคนฟัง ตอบโต้ไม่ได้
- ROS Services เป็นระบบดารสื่อสารที่คล้ายๆกับ Topics แต่จะเป็นการสื่อสารแบบสองทาง(ไป-กลับ) จะต้องมีทั้ง request กับ respone จะต้องมี action ที่ส่งไป ตามที่ request 
- topic กับ message ต่างกันตรงที่ topic เป็นชุดของข้อมูลมีลักษณะคล้ายตาราง  และtopicไม่ควรซ้ำกัน แต่ message มีลักษณะคล้ายกับ topic แต่จะเป็น ข้อมูลที่ถูกส่งผ่านระบบ ไปยังที่สั่ง
- ROS Services กับ ROS Topics ต่างกันตรงที่ การสื่อสาร ROS Services เป็นการสื่อสารแบบสองทาง(ไป-กลับ) จะต้องมีทั้ง request กับ respone จะต้องมี action ที่ส่งไป ตามที่ request  แต่ ROS Topics จะเป็นการสื่อสารแบบทางเดียว จะมีแค่ publisher กับ subscriber คล้ายทีวี มีแต่คนพูด และมีคนฟัง ตอบโต้ไม่ได้
